05
Results
Analyze performance in results/.
06
CP / Profile / Leaderboard
Track progress and compete.

Core Building Blocks

  • Pages: Each feature folder has .html + .css + .js.
  • Shared system: shared/ contains Firebase init, navigation, theme, UI helpers, and rendering logic.
  • Data layer: Firestore stores exams, questions, attempts, CP stats, and leaderboards.
  • Routing: Internal links are repo‑relative so GitHub Pages works under a subpath.
Design philosophy
Fast, clean, motivating — the UI should feel like a contest arena, not a boring form.
